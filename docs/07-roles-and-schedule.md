# 07. 역할 분담 및 일정

## 인원 및 강도

- 인원: 2명 (둘 다 일반 SW 개발 백그라운드, 하드웨어 처음, ROS2 처음)
- 기간: 4주
- 투입: 파트타임, 1인당 하루 3~4시간
- 가용 총 시간: 약 196 man-hours

## 역할 분담 (분할안 2 — 시스템/응용)

작업량 균형과 책임 분리를 위해 다음과 같이 나눈다.

### 개발자 A — 시스템 인프라

**담당 영역**
- ROS2 환경 셋업 및 다중 머신 네트워크
- FastAPI Server (rclpy 통합)
- WebSocket 브리지, REST API
- SQLite + SQLAlchemy 모델
- Whisper STT 통합
- Tailscale 셋업 + llama.cpp 클라이언트
- TTS 통합 (pyttsx3)
- 컨베이어 라즈베리파이 5 셋업 및 Flask HTTP 슬레이브
- Smart Factory Shield 핀 매핑 검증 및 모터/서보 제어 코드
- Git/CI 셋업, 배포 스크립트

**한 줄 요약**: 데이터가 어떻게 흐르는지에 책임. 백엔드와 인프라.

### 개발자 B — 응용/UX

**담당 영역**
- RealSense 카메라 + Vision Node (OpenCV 검출)
- MJPEG 스트리밍 서버
- 터틀봇3 와플 ROS2 셋업 및 동작 검증
- Nav2, slam_toolbox 통합
- 맵 매핑 및 좌표 등록
- 웹 UI (React/Vue) — 차트, 영상, 제어 버튼, 미션 패널
- 시연 시나리오 작성 및 영상 촬영
- (W4 여유 시) 4차 Dobot 통합

**한 줄 요약**: 사용자에게 보이는 부분에 책임. 비전, 로보틱스, 프론트.

### 공통 작업

- W1 D1-2: 인터페이스 문서(`11-interfaces.md`) 합의
- 매주 말 통합 테스트 및 회고
- 시연 영상 촬영 (B 주도, A 보조)
- 발표 자료 작성

## 4주 일정

매주 끝에 체크포인트가 있으며, 미달 시 다음 주에 미통과분 우선 처리하고 그 주 작업을 줄인다.

### Week 1 — 1차 기반

**목표**: 컨베이어 영상 → 검출 → 웹 UI 차트까지 연결

| 일자 | A (시스템) | B (응용) |
|---|---|---|
| D1 | ROS2 Humble 설치, 컨베이어 키트 매뉴얼 검토, 핀 매핑 확정 | ROS2 Humble 설치, RealSense 동작 확인, 인터페이스 문서 초안 |
| D2 | 인터페이스 문서 합의, FastAPI Server 골격, SQLite 스키마 | Vision Node 골격, RealSense 토픽 구독, OpenCV 빨간색 검출 |
| D3 | 컨베이어 라즈베리파이 OS/네트워크 셋업, gpiozero 동작 확인 | Vision Node 검출 결과 토픽 발행 + MJPEG 스트림 |
| D4 | 컨베이어 보드 모터/서보 제어 코드, Flask HTTP 슬레이브 | 웹 UI 골격 (Vite + React + WebSocket 연결) |
| D5 | Server에 ROS2 토픽 구독 + DB 저장 + WebSocket 푸시 | UI에 영상 표시, 검출 차트, 시작/정지 버튼 |
| D6-7 | 통합 테스트, 버그 수정 | 통합 테스트, 버그 수정 |

**체크포인트 W1말**: 컨베이어 영상이 UI에 보이고, 빨간 물체 흘리면 차트에 반영되고, 분류기가 동작.

### Week 2 — 1차 완성 + 2차 시작

**목표**: 1차 완전 동작, 터틀봇 ROS2 연결 및 단일 goal 이동

| 일자 | A (시스템) | B (응용) |
|---|---|---|

| D9 | Whisper STT 통합, 음성 업로드 REST API | 터틀봇 와플 펌웨어 업데이트, ROS2 네트워크 연결 |
| D10 | Tailscale 셋업, llama.cpp 클라이언트 + 의도 추출 프롬프트 | 노트북에서 cmd_vel로 터틀봇 수동 조작 확인 |

**체크포인트 W2말**: 1차 완성. 음성 또는 카운트 트리거 → 터틀봇이 단일 goal로 이동.

## 운영 규칙

- **매일 짧은 동기화**: 시작 또는 종료 시 5분, Discord/Slack/카톡 어디든 통일
- **주말 끝에 회고**: 30분, 체크포인트 통과 여부 + 다음 주 계획
- **인터페이스 변경은 합의**: ROS2 토픽 메시지나 WebSocket 포맷 바꾸려면 상대 동의 필요
- **막히면 2시간 룰**: 한 문제에 2시간 이상 못 풀면 상대에게 공유, 같이 본다 또는 우회
- **체크포인트는 신성함**: 미달 시 감정적으로 끌고 가지 않는다. 약속한 축소를 즉시 실행
