# 01. 프로젝트 개요

## 한 줄 정의

컨베이어 위를 흐르는 RGBY 색상 물체 중 빨간색 불량품을 비전으로 검출하고, 서보 분류기와 모바일 로봇을 연동해 자동 분류·운반하는 학습용 스마트 팩토리 시스템.

## 배경 및 학습 목표

본 프로젝트는 ROS2 분산 시스템, 컴퓨터 비전, 웹 기반 모니터링, 음성 인터페이스, 모바일 로봇 자율주행을 한 시스템 안에서 통합 경험하는 것을 목적으로 한다. 산업 자동화의 축약된 모형을 직접 만들어보면서, 각 컴포넌트가 어떻게 통신하고 협력하는지를 학습한다.

학습 산출물의 핵심은 다음과 같다:

- ROS2 Humble 분산 노드 시스템 설계 및 구현 경험
- FastAPI 기반 비동기 백엔드 + WebSocket 실시간 통신 구축
- OpenCV 색 기반 검출 알고리즘 구현 (HSV 색공간)
- 터틀봇3 SLAM 매핑 및 Nav2 자율주행 경험
- LLM 기반 음성 인터페이스 통합 (STT → 의도 추출 → ROS2 명령 → TTS 응답)
- SQLite + SQLAlchemy로 운영 데이터 영구 저장 및 조회
- 다중 PC 협업 시스템 셋업 (네트워크, 권한, 환경 변수)

## 시스템 동작 요약

- 컨베이어 벨트가 동작하며 다양한 색상의 물체가 흐른다 (RGBY 4색)
- 노트북에 연결된 RealSense D435i 카메라가 컨베이어를 위에서 촬영
- 비전 노드가 색상 분석으로 정상품(R 외 3색)과 불량품(빨간색)을 분류
- 컨베이어 끝의 서보 분류기가 갈래를 결정 — 정상은 고정 박스, 불량은 터틀봇 위 박스
- 터틀봇은 컨베이어 옆 home 위치에서 대기, 불량품 박스를 받음
- 음성 명령("불량품 비워줘") 또는 카운트 임계 도달로 출동 트리거
- 터틀봇이 SLAM 맵 위에서 적재장으로 자율 주행, 도착 후 home 복귀
- 사용자는 웹 UI에서 실시간 영상, 검출 결과 차트, 제어 버튼, STOP 안전 장치 사용
- "지금 불량률 얼마야?" 같은 자연어 질문에 시스템이 음성으로 응답

## 단계별 목표 (4단계)

본 프로젝트는 점진적 빌드를 원칙으로 한다. 앞 단계의 결과물 위에 다음 단계가 얹어지는 구조이며, 일정 압박 시 뒤 단계부터 축소·제외한다.

### 1차 — 컨베이어 비전 검출 + 서보 분류 (관통 필수)

- 컨베이어 벨트 동작 (스테퍼 모터)
- RealSense D435i로 컨베이어 위 영상 캡처
- OpenCV HSV 기반 빨간색 검출
- 검출 결과에 따라 서보 모터로 2갈래 분류 (정상/불량)
- 검출 결과 DB 저장 및 웹 UI 실시간 차트 표시

### 2차 — 터틀봇 연동 운반 (관통 필수)

- 터틀봇3 와플 ROS2 네트워크 통합
- 음성 명령 또는 누적 카운트 임계로 출동 트리거
- Nav2로 home → 적재장 자율 이동
- 도착 알림 후 home 복귀
- 미션 이력 DB 저장

### 3차 — SLAM 매핑 + 자율주행 (관통 필수)

- slam_toolbox로 현장 매핑
- 저장된 맵 위에서 Nav2 기반 자율주행
- 동적 장애물 회피 시연 및 튜닝 (시간 여유 시)
- 매핑 시연 영상 + 비교 분석 자료 작성

### 4차 — Dobot Pick & Place (도전 영역)

- 서보 분류기를 Dobot 로봇 팔로 대체
- 정지 상태 물체 Pick & Place 구현
- 움직이는 컨베이어 위 물체 추적 + 픽업 (도전)
- 시간 여유 없을 시 stretch goal로 보류

## 범위 한정 (이번 프로젝트가 다루지 않는 것)

- 영구적 학습 모델 훈련 (YOLO 커스텀 학습은 향후 과제로 보류)
- 다국어 음성 인식 (한국어 단일)
- 실제 산업 환경에서의 안전 인증 수준 검증
- 터틀봇 외 추가 모바일 로봇 협업
- 클라우드 배포, 사용자 계정/권한, 모바일 앱
- 네트워크 보안, 인증, 암호화 (로컬 신뢰 환경 가정)

## 운영 원칙

- **점진적 통합**: 모듈 하나가 단독 동작 확인된 후 다음 모듈과 결합
- **안전 우선**: 음성 명령 외에 항상 즉시 정지 가능한 물리 버튼/UI 버튼 유지
- **시연 영상 우선**: 라이브 시연 대신 사전 녹화 영상으로 결과물 보존
- **체크포인트 강제**: 매주 마감 미달 시 즉시 다음 단계 범위 축소
- **인터페이스 우선 합의**: 두 개발자가 만지는 코드 사이의 메시지 포맷은 W1 D1-2에 합의 후 고정
- **AI 도구 활용 인정**: 코드 보일러플레이트는 AI 도우미로 가속화, 디버깅·튜닝은 직접 수행
