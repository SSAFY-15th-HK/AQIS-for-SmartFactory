## 머신 간 연결 구조

```
[노트북 Ubuntu 22.04]
  ├─ USB 3.0 → RealSense D435i (컨베이어 위)
  ├─ Wi-Fi   → 컨베이어 라즈베리파이 5
  ├─ Wi-Fi   → 터틀봇3 와플 라즈베리파이
  └─ Tailscale → 집 데스크톱 (llama.cpp)

[컨베이어 라즈베리파이 5]
  └─ GPIO HAT → Smart Factory Shield → 스테퍼 + 서보 + 버튼

[터틀봇3 와플]
  └─ 내장 라즈베리파이 → OpenCR → 모터/라이다
```

## Smart Factory Shield 보드 분석

사진 분석 결과 (2024 Copyright):

**전원**
- 12VDC 입력 잭 (좌상단)
- 5V_POWER 영역 (라즈베리파이 5V 공급 또는 라즈베리파이 자체 USB-C 전원)

**액추에이터 출력**
- **SERVO**: GND, 5V, SIG 3핀 표준 서보 커넥터 (좌상단)
- **STEPPERMOTOR**: 4핀 화이트 커넥터 (1B, 1A, 2A, 2B) — 2상 스테퍼용
- **A4988 드라이버**: STEP/DIR 핀 + MS1/MS2/MS3 마이크로스텝 핀 (보라색 모듈, 방열판 부착)

**입력**
- **SWITCH1, SWITCH2**: 노란색 푸시 버튼 2개 (GPIO 직접 연결)

**라즈베리파이 GPIO 헤더 (40핀)**
- 보드 우측에 라벨 인쇄: 3V3, 5V, GP2~GP21, TXD, RXD, MOSI, MISO, SCLK, CS0, CS1, GND, SDA, SCL 등

## 라즈베리파이 5 GPIO 라이브러리 주의사항

라즈베리파이 5는 핀 컨트롤러가 RP1 칩으로 변경되어 **`RPi.GPIO` 라이브러리가 동작하지 않는다**. 다음 중 하나를 사용한다.

**`gpiozero` (권장)**
- 라즈베리파이 5에서 자동으로 새 백엔드(`lgpio`) 사용
- 추상화 수준 높음, 코드 짧음
- 본 프로젝트 표준 라이브러리로 채택

**`lgpio` (하위 레벨 필요 시)**
- 라즈베리파이 5 네이티브
- 정밀한 타이밍 필요 시

**`rpi-lgpio` (RPi.GPIO 호환 인터페이스)**
- 기존 RPi.GPIO 코드를 그대로 import만 바꿔서 사용 가능
- 키트 예제가 RPi.GPIO 기반이면 이걸로 마이그레이션

**피해야 할 것**
- `RPi.GPIO` 직접 사용 — 라즈베리파이 5에서 import 자체가 실패하거나 동작 안 함

## 카메라 설치

RealSense D435i는 컨베이어 위 약 1m 높이에 고정한다. 시야 안에 컨베이어 검출 영역 전체가 들어와야 한다.

- 노트북 USB 3.0 포트에 직접 연결 (USB 허브는 대역폭 부족 가능)
- 같은 포트에 다른 USB 3.0 장치를 함께 꽂지 않음
- 카메라 위치가 변경되면 검출 좌표 보정이 필요하므로 단단히 고정
- 4차 도전 시 카메라-Dobot 좌표 calibration이 필요하므로 위치 안정성이 중요

## 시연 환경 조건

- **조명**: 일정한 백색 LED 권장 (자연광 변화에 빨간색 검출이 흔들림)
- **공간 크기**: 5m 이상 권장, 백업으로 3x3m
- **바닥**: 평평하고 라이다 반사가 일정한 표면 (매우 검은 카펫 등은 라이다 인식 약함)
- **Wi-Fi**: 같은 SSID, 가능하면 5GHz 대역, ROS2 멀티캐스트 허용 라우터

## 전원 관리

- 컨베이어 보드: 12V DC 어댑터 (보드 잭에 직접)
- 라즈베리파이 5: 자체 USB-C 5V 27W 전원 (HAT을 통한 5V 공급은 매뉴얼 확인 후 결정)
- 터틀봇3: 와플 자체 배터리 (시연 직전 충전 확인)
- 노트북: 시연 중 어댑터 연결 권장 (배터리 모드에서 카메라 캡처 fps 떨어질 수 있음)
